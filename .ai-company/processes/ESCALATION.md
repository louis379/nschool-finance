# 問題升級流程（Escalation）

> AI 遇到不確定的事情該怎麼辦

---

## 原則

AI Agent 遇到超出自主決策權限的事情時，不應該猜測或擅自行動，而應該記錄在這裡，等人（Louis）回覆後再執行。

## 升級判斷流程

```
AI 遇到決策點
    │
    ├── 在自主權限內？ ──→ 是 ──→ 直接執行，記錄到對應文件
    │
    └── 不在自主權限內？
         │
         ├── 緊急程度？
         │    ├── Production 掛了 ──→ 先做最小修復，同時升級
         │    └── 非緊急 ──→ 記錄到本文件，等待回覆
         │
         └── 記錄格式見下方
```

## 需要升級的事項

以下事項 **必須** 升級給人確認：

1. **商業邏輯變更** — 任何影響用戶看到的數據計算方式
2. **定價/訂閱策略** — 任何涉及金錢的決策
3. **對外溝通** — 任何公開發佈的內容
4. **大型架構重構** — 影響 3 個以上模組的改動
5. **資料庫 schema 變更** — 新增/刪除/修改欄位
6. **新的第三方服務** — 整合新的外部 API 或工具
7. **部署到 Production** — 所有部署都需要確認
8. **刪除功能** — 移除任何現有功能
9. **安全相關** — 任何涉及 Auth、權限、加密的變更

## 升級紀錄格式

```markdown
### [ESC-XXX] 標題

- **日期**：YYYY-MM-DD
- **提出者**：AI Agent（哪個部門）
- **緊急度**：高/中/低
- **問題描述**：[遇到什麼問題]
- **AI 的建議方案**：[AI 認為應該怎麼做]
- **備選方案**：[其他可能的做法]
- **等待決策**：[需要人決定什麼]
- **狀態**：待回覆 / 已回覆 / 已執行

#### 人的回覆：
（Louis 在這裡寫回覆）
```

## 回覆指引
