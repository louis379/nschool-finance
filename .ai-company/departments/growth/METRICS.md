## 關鍵指標定義

### 用戶活躍

```
DAU（日活躍用戶）= 當日有開啟 App 的獨立用戶數
WAU（週活躍用戶）= 過去 7 天有完成學習/交易的獨立用戶數
MAU（月活躍用戶）= 過去 30 天有任何行為的獨立用戶數

Stickiness = DAU / MAU（目標 > 20%）
```

### 留存率

```
D1 留存 = 第 1 天回訪用戶 / 第 0 天新用戶
D7 留存 = 第 7 天回訪用戶 / 第 0 天新用戶
D30 留存 = 第 30 天回訪用戶 / 第 0 天新用戶
```

### 學習參與度

```
課程開始率 = 開始課程 / 看到課程推薦
課程完成率 = 完成課程 / 開始課程（目標 > 60%）
每用戶課程數 = 總完成課程 / MAU
```

### 模擬交易參與度

```
交易啟動率 = 有過第一筆交易 / 總用戶
交易頻率 = 月均交易次數 / 活躍交易用戶
```

## 商業指標（上線後追蹤）

| 指標 | 計算方式 | 目標 |
|------|---------|------|
| 付費轉換率 | 付費用戶 / 總用戶 | > 5% |
| ARPU | 月總收入 / MAU | > NT$150 |
| LTV | ARPU × 平均訂閱月數 | > NT$1,800 |
| CAC | 行銷費用 / 新用戶數 | < NT$150 |
| LTV/CAC | 長期價值 / 獲客成本 | > 3x |

## A/B 測試計畫

### 實驗 #001：首次使用體驗
**假設：** 強制用戶完成投資程度問卷，比直接進首頁有更高的 D7 留存
**控制組：** 直接進首頁
**實驗組：** 3 題問卷 → AI 個人化首頁
**成功指標：** D7 留存率提升 5%+
**所需樣本：** 各 200 用戶
**狀態：** 🔲 等待上線後執行

### 實驗 #002：每日提醒時間
**假設：** 晚上 8-9 點的推播比早上提醒有更高的點擊率
**成功指標：** 推播點擊率 > 15%
**狀態：** 🔲 等待推播功能完成
