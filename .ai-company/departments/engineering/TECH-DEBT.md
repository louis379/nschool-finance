## 格式

```markdown
### [TD-XXX] 標題

- **嚴重度**：高/中/低
- **影響範圍**：[哪些模組/功能受影響]
- **描述**：[問題描述]
- **建議方案**：[如何解決]
- **預估工時**：[時間]
- **狀態**：待處理 / 進行中 / 已解決
```

## 高嚴重度

### TD-001 缺少全域錯誤邊界（Error Boundary）

- **嚴重度**：高
- **影響範圍**：整個應用程式
- **描述**：目前沒有 React Error Boundary，任何元件拋出錯誤都會導致整頁白屏。
- **建議方案**：在 Layout 層加入 Error Boundary，搭配友善的錯誤頁面。
- **預估工時**：2 小時
- **狀態**：待處理

### TD-002 API 路由缺少統一錯誤處理

- **嚴重度**：高
- **影響範圍**：所有 API endpoints
- **描述**：各 API 路由的錯誤處理方式不一致，部分路由缺少 try-catch。
- **建議方案**：建立統一的 API middleware 處理錯誤回應格式。
- **預估工時**：3 小時
- **狀態**：待處理

## 中嚴重度

### TD-003 元件缺少 loading/skeleton 狀態

- **嚴重度**：中
- **影響範圍**：儀表板、交易頁
- **描述**：資料載入時顯示空白，用戶體驗不佳。
- **建議方案**：為資料密集的元件加入 Skeleton 載入狀態。
- **預估工時**：4 小時
- **狀態**：待處理

### TD-004 硬編碼字串散落各處

- **嚴重度**：中
- **影響範圍**：所有 UI 文字
- **描述**：UI 文字直接寫在元件中，未來做多語系時會很痛苦。
- **建議方案**：集中管理 UI 文字（可先用簡單的 constants 檔案）。
- **預估工時**：6 小時
- **狀態**：待處理

## 低嚴重度

### TD-005 部分 CSS 可改用 TailwindCSS

- **嚴重度**：低
- **影響範圍**：樣式一致性
- **描述**：早期開發有些地方用了 inline style 或 CSS modules，可統一為 Tailwind。
- **建議方案**：逐步遷移，不急。
- **預估工時**：持續性
- **狀態**：進行中（隨開發順手改）
